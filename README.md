# Ethereum Payment App

This project is a decentralized application (DApp) built to track debit and credit among a group of people, functioning as a blockchain version of the Splitwise app. It allows users to track who owes whom money without relying on a trusted third party.

## Overview

The system is designed to be highly efficient, minimizing the gas cost and storage required on the blockchain. No actual value is transferred on the blockchain; ether is only used to pay for gas. The public on-chain storage serves as a single source of truth for debts.

A key feature of this application is the automatic resolution of debt cycles. If a cycle of debt forms (e.g., User A owes User B, User B owes User C, and User C owes User A), the application resolves the cycle by subtracting the minimum debt amount from every step in the cycle. 

## Architecture

The project consists of two main components:
* **Smart Contract:** Written in Solidity (v0.8.17) and deployed on the blockchain. The contract (`contracts/mycontract.sol`) stores the state and exposes functions like `lookup` and `add_IOU`.
* **Web Client:** Written in HTML and JavaScript using the ethers.js library. The client (`web_app/script.js` and `web_app/index.html`) observes the blockchain and interacts with the smart contract. It includes functions like `getUsers`, `getTotalOwed`, `getLastActive`, and performs the Breadth-First Search (BFS) to find debt cycles before submitting IOUs.

## Setup and Installation

### 1. Install Dependencies
This project uses Foundry for building and deploying the smart contracts. Unix-based systems or Windows users (via GIT BASH/WSL) can install Foundry using the following commands:

bash
curl -L https://foundry.paradigm.xyz | bash
foundryup


### 2. Start the Local Node
Start a local Ethereum node using Anvil by running the following command in your terminal:

bash
anvil


Leave this terminal running. It will output a list of available accounts and private keys.

### 3. Build and Deploy
Open a new terminal window and navigate to the project directory. Run the following command to deploy the contract, replacing `[PRIVATE KEY HERE]` with one of the keys generated by Anvil:

bash
forge create ./contracts/mycontract.sol:Splitwise \
 --rpc-url http://127.0.0.1:8545 \
 --private-key [PRIVATE KEY HERE] \
 --broadcast


Note the deployed contract address output in the terminal.

### 4. Configure the Web Client
Update the frontend to communicate with your newly deployed contract:
1. Open `web_app/script.js`.
2. Update the contract address variable with the address from the previous step.
3. Extract the ABI from the auto-generated `build/mycontract.sol/Splitwise.json` file. Copy the array starting from the square bracket after the 'abi' field and update the ABI variable in `script.js`.

### 5. Run the Application
You can access the frontend by running a local web server in the `web_app` directory. For example, using Python:

bash
cd web_app
python -m http.server 8000


Open a web browser and navigate to `http://localhost:8000`.

## Testing

The project includes a built-in `sanityCheck()` function in the `script.js` file to verify core functionality. 

To run the test:
1. Open the browser's Developer Console.
2. Ensure you are working with a fresh state by restarting Anvil and redeploying your contract.
3. Call `sanityCheck()` directly in the console.
